Capabilities

Three Ways to Capture Requirements

Choose the intake mode that fits your workflow. All modes produce structured Draft PRDs ready for elaboration.

Chat-Based Intake

Conversational requirements capture guided by 5 core prompts. Claude asks one focused question at a time, streaming responses in real-time. The Draft Extractor runs in parallel to build your PRD.

Visual Intake

Open any URL in a real Chrome browser via CDP. Navigate naturally, log in via SSO, then enable Inspect mode. Click any UI element to capture metadata and describe your desired change.

Repository Intake

Point to a GitHub repository. The system analyzes the codebase structure, README, and architecture to pre-fill your intake draft with existing context.

Live Draft PRD

A terminal-themed side panel shows your structured draft updating in real-time. Readiness score, completeness bar, and all fields organized by the 5 core prompts framework.

Project Memory

Durable facts, constraints, and preferences persist across sessions. Claude never re-asks questions you've already answered. Rolling summaries keep long sessions fast.

Search & Restore

Find and reopen any prior intake session. Search across workspaces, messages, drafts, and memory items. Full session restore with context continuity.

Agent Pipeline

What Happens on Every Message

Six agents run in a coordinated pipeline to capture, structure, and remember your requirements.

Context Assembly

Load recent messages, current draft, rolling summary, active memory items, and open questions into a runtime context bundle.

Tool Planning

Claude decides whether MCP tools (template discovery, standards lookup, integration discovery) would add value for this turn.

Claude Response

Stream a focused, single-question response via Vertex AI. Context-aware — never re-asks answered questions.

Draft Extraction

In parallel, extract structured data into the Draft PRD. Compute weighted readiness score across the 5 core prompts.

Memory Curation

Extract durable facts (constraints, preferences, standards) and persist them for future sessions.

Summary Refresh

Every 5 turns or when readiness changes significantly, generate a rolling summary that replaces full transcript replay.
